FIELD CLASSES & FALLBACK RULES
================================================================================

1. GOVERNING ENGINEERING FIELDS (11 fields)
   Examples: project_design_standard, frame_type, grid_spacing, bolt_grade
   Fallback: DWG → STAAD → PDF (NO history, NO guessing)
   Blocking: YES — if missing, generation STOPS
   Confidence: 0.8+ required for Governing fields
   Prohibited: History, templates, user guessing, defaults

2. DERIVED ENGINEERING FIELDS (6 fields)
   Examples: buildup_weight, bolt_projection, member_marks, grid_line_count
   Fallback: AUTO-CALCULATION from governing fields ONLY
   Blocking: YES — if calculation fails or missing inputs, BLOCK
   Confidence: 0.9+ for derived fields
   Prohibited: Override, history, manual adjustment without recalc

3. PRESENTATION/TEMPLATE FIELDS (6 fields)
   Examples: output_class, sheet_size, drawing_title, revision_number
   Fallback: Template → History → User Default (ALL OK)
   Blocking: NO — missing values just use defaults
   Confidence: 0.5+ acceptable for presentation
   Prohibited: None — fallback fully permitted

4. METADATA FIELDS (6 fields)
   Examples: extraction_timestamp, extraction_method, confidence_score, conflict_flag
   Fallback: System auto-generated; NO user override
   Blocking: NO — system fills defaults
   Confidence: N/A — system-calculated
   Prohibited: User override of confidence score

5. CONTROL/STATUS FIELDS (5 fields)
   Examples: job_status, release_gate_status, manual_review_flag, override_pending
   Fallback: System rule evaluation → User action → Default
   Blocking: YES (if rule says BLOCK, respect it)
   Confidence: N/A — rule-determined
   Prohibited: None — system governs

6. OUTPUT-ONLY FIELDS (5 fields)
   Examples: dxf_grid_line_count, dwg_conversion_status, pdf_generation_status
   Fallback: GENERATED FRESH during output phase; NO CACHE
   Blocking: YES — if generation fails, output STOPS
   Confidence: N/A — fresh generation only
   Prohibited: Cached values, previous generation results, any override

BLOCKING CONDITIONS (HARD STOPS)
================================================================================

S1 (INTAKE):
  ✗ project_design_standard missing & cannot infer
     → Error: "Design standard mandatory for all section/bolt lookups"
  ✗ Archive > 25 files
     → Error: "Escalate for manual file selection"

S2 (DESIGN PARSING):
  ✗ grid_spacing_x or grid_spacing_y missing from ALL sources
     → Error: "Grid spacing required for GA generation"
  ✗ DWG entity type validation fails (VR-PARSE-01)
     → Error: "Entity types invalid for design standard"
  ✗ Grid line count ≠ bay count (VR-PARSE-02)
     → Error: "Grid geometry mismatch"

S3 (FIELD POPULATION):
  ✗ frame_type missing and cannot be inferred
     → Error: "Frame type drives connection/crane/grid validation"
  ✗ Built-up section flag = TRUE but plate dimensions missing
     → Error: "Plate dimensions mandatory for built-up members (VR-NEW-02)"
  ✗ Confidence < 0.4 on any governing field
     → Error: "Confidence threshold too low for generation"
  ✗ Unresolved multi-source conflict (sources disagree, no quorum)
     → Error: "Multiple sources conflict; engineer review required"
  ✗ All fallback sources exhausted (VR-FALLBACK-01)
     → Error: "No controlled secondary source available"

S4 (AB OUTPUT):
  ✗ connection_type missing for structural frame
     → Error: "Connection type required for anchor bolt generation"
  ✗ bolt_grade or bolt_size missing
     → Error: "Bolt specification cannot be determined"
  ✗ Grid line count mismatch in generated DXF
     → Error: "DXF grid geometry invalid (VR-DXF-01)"
  ✗ DXF generation failure
     → Error: "ezdxf generation error (VR-OUTPUT-01)"

S5 (GA OUTPUT):
  ✗ Grid spacing tolerance exceeded (> ±2mm)
     → Error: "Grid spacing validation failed (VR-DXF-02)"
  ✗ Member mark coverage incomplete
     → Error: "Generated marks ⊂ database marks (VR-DXF-03)"
  ✗ PDF generation failure
     → Error: "LibreOffice PDF conversion failed (VR-OUTPUT-03)"

S6-S10 (RELEASE):
  ✗ Traceability incomplete
     → Error: "Not all fields have traceability entry (VR-TRACE-01)"

================================================================================
MANUAL REVIEW TRIGGERS (PAUSE & REVIEW)
================================================================================

P2 ENGINEER REVIEW REQUIRED:
  ⚠ Confidence 0.5-0.6 on governing field
     → Action: Engineer validates value or rejects
     → Result: Update confidence, approve, or escalate
     
  ⚠ Multi-source conflict with quorum (2+ sources agree)
     → Action: Engineer validates consensus is correct
     → Result: Approve or escalate conflicting source
     
  ⚠ Derived field conflict (calculated ≠ extracted)
     → Action: Engineer confirms calculation is correct
     → Result: Approve calculation; note extracted error
     
  ⚠ Built-up member with missing detail
     → Action: Engineer defines missing plate dimensions
     → Result: Proceed with engineer-provided values
     
  ⚠ Connection type inferred from forces (confidence < 0.65)
     → Action: Engineer confirms connection is appropriate
     → Result: Proceed or update to correct type
     
  ⚠ Incomplete source with limited alternatives
     → Action: Engineer evaluates data; judgment call
     → Result: Proceed, escalate, or request more data

APPROVAL REQUIRED: P2 Engineer sign-off (not auto-approved)

================================================================================
WARNING-ONLY CONDITIONS (CONTINUE GENERATION)
================================================================================

⚠ Presentation field missing (drawing_title, revision_number)
  → Use system default, continue generation

⚠ Format normalization needed (whitespace, case)
  → Auto-fix, continue generation

⚠ PDF data with confidence 0.5-0.7 (secondary source)
  → Log confidence, use with caution, continue generation

⚠ Metadata missing (timestamps, extraction_method)
  → System auto-generates defaults, continue

⚠ Historical value used for presentation field (allowed fallback)
  → Log fallback source, confidence = 0.50, continue

EFFECT: Logged as warning, confidence scored, but generation proceeds

SPECIAL FALLBACK CASES
================================================================================

CASE 1: INCOMPLETE STAAD .std FILES

Symptom: STAAD has section defined but properties incomplete
Fallback Chain:
  DWG → Extract member properties (confidence ≥ 0.75)
  IF DWG unavailable → Section DB lookup
  IF DB lookup fails → BLOCK, manual review

Symptom: STAAD bolt table empty or incomplete
Fallback Chain:
  DWG → Extract bolt pattern from anchor coords (confidence ≥ 0.8)
  IF DWG unavailable → PDF OCR (confidence ≤ 0.5, needs review)
  IF all sources fail → BLOCK, error "Anchor bolt data missing"

Symptom: STAAD connection definition missing
Fallback Chain:
  connection_type (F-157) explicit or derived?
  DWG → Extract from connection blocks
  STAAD → Connection definition
  Infer from forces (confidence 0.6, flag for review)
  DB default (confidence 0.5, manual review required)

CONFIDENCE SCORING:
  Section + properties complete from STAAD: 0.95
  Section + properties supplemented from DWG: 0.80
  Section + properties supplemented from DB: 0.75
  Bolt data complete from STAAD: 0.95
  Bolt data extracted from DWG: 0.75
  Bolt data extracted from PDF: 0.45

---

CASE 2: MBS (LEGACY DATABASE) MISSING FIELDS

Symptom: MBS field is NULL or empty
Action: Treat as missing, apply field class fallback rules

Symptom: MBS field is outdated (> 6 months)
Action: Don't use for governing fields; confidence < 0.6

Symptom: v2 field deprecated, must map to v2.1
Mapping Examples:
  MBS.member_code → F-070 (member_section) [Parse code → lookup]
  MBS.connection_standard → F-157 (connection_type) [Parse → enum]
  MBS.bolt_standard → F-084, F-085 [Parse → split fields]

Action: Apply transformation; if confidence < 0.6 → manual review

Symptom: MBS field has no v2.1 equivalent
Action: If governing field → try to infer or BLOCK
        If presentation field → skip, use template

CONFIDENCE SCORING:
  MBS complete, current (< 6 months): 0.70
  MBS incomplete or outdated: 0.40
  MBS transformed from deprecated field: 0.55
  MBS with no v2.1 mapping: DO NOT USE for governing

---

CASE 3: ETABS / PROTA PARTIAL DATA

Symptom: ETABS section defined; no fabrication detail
Fallback: DWG primary, DB lookup secondary
Confidence: ETABS analysis = 0.50 (low)
           + DWG supplement = 0.80
           + DB lookup = 0.75

Symptom: ETABS forces show connection required but undefined
Action: Infer connection type from force magnitude
  High forces → Moment connection
  Low forces → Simple connection
Confidence: Force-inferred = 0.65 (needs review)

Symptom: ETABS analysis includes partial member set
Action: Check if expected (e.g., analyzed beam subset)
        If YES: proceed with available data
        If NO: BLOCK (missing critical members risk)

CONFIDENCE SCORING:
  Complete section with fabrication: 0.85
  Section only, no fabrication: 0.50
  Connection inferred from forces: 0.65
  Partial member set: BLOCK if critical

---

CASE 4: MISSING CONNECTION DETAILS

Fallback Chain for connection_type (F-157):
  1. DWG block analysis (confidence 0.85) ✓
  2. STAAD definition (confidence 0.80) ✓
  3. Infer from forces (confidence 0.60) ⚠ Review
  4. DB default for frame type (confidence 0.50) ⚠ Review
  5. All sources exhausted → BLOCK ✗

Fallback Chain for bolt details (F-084, F-085):
  1. DWG extraction (confidence 0.80) ✓
  2. STAAD bolt table (confidence 0.75) ✓
  3. PDF schedule (confidence 0.70) ⚠ Review
  4. DB lookup by connection type (confidence 0.65) ⚠ Review
  5. All sources exhausted → BLOCK ✗

Fallback Chain for bolt_projection_above_ffl (F-192):
  1. DWG anchor coords (confidence 0.80) ✓
  2. Calculate from FFL + member + bolt length (confidence 0.75) ⚠
  3. DB default (confidence 0.50) ⚠ Review
  4. All sources exhausted → BLOCK ✗

---

CASE 5: MISSING BUILT-UP SECTION DATA

Symptom: Built-up section flag = TRUE; plate dimensions missing
Blocking: VR-NEW-02 triggers (Release-Blocker)
Action: All of F-184 to F-189 MANDATORY; missing = BLOCK

Fallback Chain:
  1. DWG section detail (confidence 0.80) ✓
  2. STAAD section comment (confidence 0.70) ✓
  3. PDF technical drawing (confidence 0.55) ⚠ Review
  4. All sources exhausted → BLOCK ✗

Symptom: Plate material grade missing
Fallback Chain:
  1. DWG material note (confidence 0.75) ✓
  2. STAAD material specification (confidence 0.70) ✓
  3. Design standard default (confidence 0.50) ⚠ Review
  4. All sources exhausted → NULL (may block validation)

Symptom: Web-flange connection (bolts/welds) unspecified
Fallback Chain:
  1. DWG detail view (confidence 0.80) ✓
  2. STAAD connection sheet (confidence 0.70) ✓
  3. Assume standard for size (confidence 0.55) ⚠ Review
  4. All sources exhausted → BLOCK ✗
