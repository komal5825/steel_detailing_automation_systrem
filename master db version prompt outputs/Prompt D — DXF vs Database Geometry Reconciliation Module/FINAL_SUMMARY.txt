❓ FAQ
═══════════════════════════════════════════════════════════════════════════════

Q: How long will implementation take?
A: 6 weeks (390 hours). Can parallelize some weeks if needed.

Q: Do I need to change Prompt B or Prompt C?
A: No. Prompt D is additive; integrates with existing prompts B & C.

Q: What if geometry validation finds a mismatch?
A: Check geometry_check_result_log. Severity determines action:
   - CRITICAL: STOP (escalate to P3 principal engineer, 4-hour SLA)
   - HIGH: HOLD (P2 review, 24-hour SLA)
   - MEDIUM: WARN (log + proceed)
   - LOW: PASS (automatic)

Q: Can I override geometry validation?
A: Yes, with P3 principal engineer approval + documented exception.

Q: Which tolerances should I use?
A: Look up your design_standard in geometry_tolerance_master table.
   All tolerances based on AISC 360, AWS D1.1, or internal standard.

Q: What happens if DXF doesn't match database?
A: Reconciliation detects mismatch, classifies severity, queues for review.
   Fallback chain: If DXF has CRITICAL/HIGH mismatch, skip DXF; try STAAD.
