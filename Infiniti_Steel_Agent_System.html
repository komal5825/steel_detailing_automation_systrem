Section 01

Executive Summary

This document defines the complete system architecture for the Infiniti Solutions Steel Detailing Multi-Agent Automation System — a local-first, desktop-based platform that reads engineering design files and produces fully validated AB drawings, GA drawings, shop/detail drawings, sheeting, shipping lists, installation drawings, and final release packages in DWG and PDF formats.

Governing Principle
The design file is the absolute source of truth for all engineering values. Historical job data is used exclusively for template learning, field mapping, and pattern recognition. Historical data must never override live design file values.
PHASE 1 — ONE-TIME + PERIODIC
Master Database Creation
50 reference jobs → Template library, field dictionary, source mapping rules, validation rules. Foundation for all agents.
PHASE 2 — PER PROJECT
Design Ingestion & AB/GA Layer
Ingestion → Completeness check → AB generation → GA generation → AB/GA cross-validation. Sequential gate-by-gate.
PHASE 3 — PER PROJECT
Detailing & Release Layer
Sheeting → Shop drawings → Shipping list → Installation → Final supervisor check → Output generation.

Section 05

Phase 1 — Master Database Creation

Trigger Logic
Initial: Manual one-time run by IT team on the 50 reference jobs.
Periodic: System auto-flags after every 10 new approved projects. Engineer manually triggers re-analysis. LLM proposes rule diffs. Engineer approves or rejects before any rule update is applied.

Section 06

Phase 2 — AB/GA Execution Detail

Critical: AB Before GA (Sequential)
AB drawing contains 3 views: Plan (bolt layout, grid, quantities), Section (embedment depth, base plate, grout), Detail (bolt assembly). GA plan view directly uses AB grid positions. AB approval is a hard prerequisite for GA start. If AB is re-generated, GA must regenerate too.

P2-01Design File IngestionRevision conflict: engineer selects governing fileP2-02 blocked if not PASS

P2-03AB GenerationFallback escalation after 2 failuresP2-04 hard blocked until AB approved
P2-04GA GenerationFallback escalation after 2 failuresP2-05 blocked if not PASS
P2-05AB/GA ValidationCritical discrepancy reviewALL Phase 3 blocked on FAIL
Section 10

Three-Level Learning System

LevelNameMechanismHuman RoleApplies To
Level 1Rule RefinementReads correction_events. 5+ same-field errors → proposes rule update → rule_proposals tableReviews rule diff. Approves or rejects. No auto-apply.validation_rules table
Level 2Smart SuggestionsChromaDB similarity search on missing fields. Suggests value from similar past jobs.UI popup: Accept or Reject per field. Accepted = engineer-approved.Per-project field completion
Level 3Pattern LearningApproved project → vector embedding → ChromaDB. New project → similarity search → template suggestions.Reviews template suggestions. Still selects final template family.Template & layout guidance only
Hard Rule — All Three Levels
No level of the learning system may override design file engineering values. All learning is confined to: template layout, field value suggestions (human-approved), and rule updates (human-approved).
